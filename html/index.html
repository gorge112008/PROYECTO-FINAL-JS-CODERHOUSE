PROYECTO FINAL

*Descripción del Proyecto:

El proyecto se basa en la creación de un Simulador Interactivo de Venta de comida rápida.

  • El simulador cuenta con las siguientes funcionalidades:

    LOGIN:
    =>Permite el ingreso a la aplicación en forma de invitado o mediante un logueo.
    =>El logueo puede darse por solo una sesión o se puede almacenar la sesión mediante un checkbox.
    =>Para ingresar primero se debe hacer un registro y automaticamente se habilita la opción de ingreso.
    =>Si se activa el checkbox de recordar, el usuario seguirá registrado después de cerrar sesión y si se ingresa directamente, se podrá acceder a su sesión.
    =>Al tener una sesión guardada, en el login se habilita la opción de nuevo, esto elimina el antiguo login y permite una nueva sesión.
    =>Muestra notificaciones en caso de dejar algún campo en blanco, al ingresar al login un usuario previamente registrado, al ingresar mal la contraseña de un usuario registrado.
    SESIÓN:
    =>Permite el recorrido de opciones del simulador mediante una barra de navegación.
    =>Muestra los menús disponibles de forma detallada.
    =>Permite el ingreso de un nombre para el cliente atendido, caso contrario se le tomará como anónimo.
    =>Permite seleccionar un filtrado de menús para distintas carteras.
    =>Muestra la cantidad de menús disponibles en base al stock disponible.
    =>Habilita la opción de añadir menús al pedido mediante selección luego de seleccionar el tipo de menú.
    =>Habilita la opción de cancelar menús seleccionados en base a la cantidad pedida luego de añadir como mínimo un menú.
    =>Deshabilita la opción de cancelar menús si se cancelan todos los menús del pedido.
    =>Brinda una Pre-visualización del pedido detallado antes de finalizar el pedido.
    =>Permite la finalización del pedido y visualización final del pedido detallado.
    =>Permite realizar un nuevo pedido, reiniciando el stock.

Nota importante:

Para que se ejecute el código correctamente es necesario abrirlo desde live server.
